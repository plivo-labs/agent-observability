Agent Observability captures what your voice and chat agents actually do —
session recordings, per-turn latency, token usage, tool calls, and eval
verdicts — and gives you a dashboard to inspect it all. It receives session
reports from the agent-transport SDKs, stores them in Postgres, and serves a
dashboard UI you can run as-is or rebuild from the published component
registry.

## How the pieces fit

The [server](/server/quickstart) is the ingest endpoint and the dashboard
host. The [Python](/sdks/python/quickstart) and
[Node](/sdks/node/quickstart) SDKs instrument your agents to send it session
reports; the [pytest](/eval-plugins/pytest) and
[Vitest](/eval-plugins/vitest) plugins stream eval results from your test
runs. Everything you see in the dashboard is built from the published
[UI component registry](/ui/provider) — mount the provider, then compose the
[hooks](/ui/hooks/use-sessions), [pages](/ui/pages/sessions-page),
[components](/ui/components/metric-summary-cards), and
[charts](/ui/charts/latency-percentiles) to build exactly the views you
need.
